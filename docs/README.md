# OMX 배송로봇 문서

이 디렉터리는 한국지능로봇경진대회 배송로봇의 기획, 구현 진행, 시험 결과를 관리하는 기준 문서 공간이다. 패키지 README는 설치와 실행 방법만 설명하고, 프로젝트 전체 결정과 진행 상태는 이 디렉터리에서 관리한다.

## 문서 목록

| 문서 | 역할 | 상태 |
|---|---|---|
| [최종 프로젝트 기획 및 진행 계획](<./최종 계획서.md>) | 전체 구조, 임무 순서, 현재 진행, 단계별 완료 기준 | 기준 문서 |
| [UWB 위치 제어 계획서](<./UWB 위치 제어 계획서.md>) | DWM1000 입력, 위치 추정, 목표점 제어 상세 계획 | 작성 완료, 구현 대기 |
| [OMX EEF 비전 계획서](<./OMX EEF 비전 계획서.md>) | ArUco 인식, 카메라·TF 보정, 정확도 시험 계획 | 작성 완료, 검증 대기 |

| `통합 시험 결과.md` | 이동-파지-배송-배치 전체 반복 시험 결과 | 작성 예정 |

## 상태 표기

| 상태 | 의미 |
|---|---|
| `완료` | 코드와 시험 근거가 모두 있음 |
| `부분 완료` | 일부 코드가 있으나 최종 계약 또는 통합 시험이 남음 |
| `구현 대기` | 설계만 확정되고 실행 코드는 없음 |
| `검증 대기` | 코드는 있으나 실기기 또는 반복 시험이 남음 |
| `보류` | 선행 조건이나 외부 결정이 필요함 |

진행 상태를 `완료`로 바꿀 때는 사용한 설정, 실행 명령, 결과 로그 또는 표를 같은 문서에 남긴다. 시뮬레이션 성공과 실기기 성공은 별도로 기록한다.

## 문서 관리 원칙

- 프로젝트 기준 로봇은 **TurtleBot3 Waffle Pi + OpenMANIPULATOR-X**로 통일한다.
- 최종 통합 실행 진입점은 `turtlebot3_control/launch/omx_turtle.launch.py`로 유지한다.
- 모델 파일, 파라미터, 마커 크기, 좌표계가 바뀌면 문서의 기준값과 checksum을 함께 갱신한다.
- 계획값은 `예정`, 측정값은 시험 조건과 단위를 명시한다.
- 패키지별 상세 설계가 전체 기획과 충돌하면 `최종 계획서.md`를 먼저 갱신한 뒤 패키지 문서를 맞춘다.
